## Overview

The Toit project contains the Toit programming language implementation. It consists of the compiler, virtual machine, and standard libraries that together enable Toit programs to run on the ESP32 and other embedded devices, as well as on host machines (Linux, macOS, Windows).

## Components

### Compiler & Tools
- **Purpose:** Compiles Toit code into snapshots, serves as the local package manager, and provides the Language Server Protocol (LSP) backend for IDE integration.
- **Location:** `src/compiler/`

### Virtual Machine
- **Purpose:** Executes Toit snapshots. Contains the interpreter, heap management, scheduler, and primitive implementations. It integrates smoothly into embedded hardware, specially targeting ESP32.
- **Location:** `src/` (and platform specific implementations like `vm_esp32.cc`, `vm_linux.cc`)
- **Dependencies:** `mbedtls`, `lwip`, `freertos` (on ESP32).

### Standard Library
- **Purpose:** The core libraries written in the Toit language that provide functionality (e.g., networking, math, IO) atop the VM primitives.
- **Location:** `lib/`

## Data Flow
1. **Compilation:** Toit CLI compiles `.toit` source files into a `.snapshot` format.
2. **Packaging:** Snapshots can be embedded into firmware envelopes.
3. **Execution:** The Toit VM loads the `.snapshot` (or boot snapshot) and executes the bytecodes.

## Conventions
- **Naming:** C++ files use `snake_case`. Standard C++ paradigms typically used. `.toit` files use idiomatic Toit conventions.
- **Structure:** Modular C++ and Toit codebases separated clearly by directories (`src` for C++, `lib` for Toit).
- **Testing:** Contains a robust `tests/` directory covering compiler semantics and VM executions.
